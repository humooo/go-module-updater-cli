# go-module-updater-cli

CLI-утилита на Go: клонирует указанный **git-репозиторий**, читает **`go.mod`** и выводит **имя модуля**, **версию Go** из директивы `go` и **список зависимостей, для которых доступно обновление** (через `go list -m -u`).

---

## Возможности

- Shallow clone (`--depth=1`, одна ветка) во временную директорию с последующей очисткой  
- Парсинг **`module`** и **`go`** из корневого **`go.mod`**  
- Анализ обновлений: **`go list -m -u -json all`**, в выводе только модули с полем **`Update`**  
- Таймауты на **`git clone`** и **`go list`** (см. константы в `main.go`)  
- Неинтерактивный режим для git/go: **`GIT_TERMINAL_PROMPT=0`**  
- Текстовый вывод или **`--json`**

---

## Требования

| Зависимость | Назначение |
|-------------|------------|
| **Go** ≥ версии из [`go.mod`](go.mod) | сборка и запуск `go list` |
| **git** | клонирование репозитория |
| Сеть | `git clone`, загрузка модулей при `go list` |

Анализ выполняется **локально установленным** тулчейном Go. Строка **`go`** в выводе берётся из **`go.mod`**; результат `go list` может отличаться на разных версиях Go — для воспроизводимости зафиксируй версию в CI или в документации.

## Установка

```bash
git clone https://github.com/humooo/go-module-updater-cli.git
cd go-module-updater-cli
go build -o gmuc .
```

Или без бинарника:

```bash
go run . -- <git-repo-url>
```

---

## Использование

```text
gmuc [--json] <git-repository-url>
```

| Флаг | Описание |
|------|----------|
| `--json` | Печатать результат в JSON в **stdout** (удобно для скриптов) |

Пример:

```bash
./gmuc https://github.com/golang/example
./gmuc --json https://github.com/golang/example
```

### Пример текстового вывода

```text
module: golang.org/x/example
go: 1.23.0
updates:
  - golang.org/x/tools: v0.33.0 -> v0.43.0 (direct)
  - golang.org/x/net: v0.40.0 -> v0.52.0 (indirect)
  ...
```

Если обновлений нет:

```text
updates:
  (none)
```

### Пример JSON (`--json`)

```json
{
  "module": "golang.org/x/example",
  "goVersion": "1.23.0",
  "updates": [
    {
      "path": "golang.org/x/tools",
      "currentVersion": "v0.33.0",
      "latestVersion": "v0.43.0",
      "indirect": false
    }
  ]
}
```

Ошибки и подсказки по использованию пишутся в **stderr**; полезный результат — в **stdout**.

## Как это устроено

1. **`git clone`** репозитория во временный каталог  
2. Проверка наличия **`go.mod` в корне** клона  
3. **`modinfo`**: разбор **`go.mod`** → путь модуля и **`go`**  
4. **`go list -m -u -json all`** в каталоге модуля  
5. **`updates`**: разбор потока JSON, отбор записей с **`Update`**, без главного модуля (`Main`)

```text
repo URL → clone → go.mod → module + go version
                    ↓
              go list -m -u -json all → фильтр по Update → вывод
```

---

## Ограничения

- Поддерживается **один** корневой **`go.mod`** в корне репозитория (без поиска в подпапках / `go.work` в этой версии не обрабатывается).  
- Расчёт на **публичные** репозитории; для приватных модулей нужны **`GOPRIVATE`**, credentials и корректная среда — иначе `git` / `go` могут зависнуть или запросить ввод (частично снижено через **`GIT_TERMINAL_PROMPT=0`**).  
- Очень большие репозитории: clone может занять много времени; действуют **таймауты** на clone и `go list`.

## Коды выхода (кратко)

| Код | Ситуация |
|-----|----------|
| `0` | Успех |
| `1` | Ошибка записи JSON |
| `2` | Неверные аргументы (usage) |
| `3` | Не удалось создать временную директорию |
| `4` | Ошибка / таймаут `git clone` |
| `5`–`7` | Проблемы с `go.mod` на диске |
| `8` | Ошибка парсинга `go.mod` |
| `9` | Ошибка / таймаут `go list` |
| `10` | Ошибка разбора вывода `go list` |
